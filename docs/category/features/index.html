The expo-router Link component supports client-side navigation to a route. It is similar to the Link component in react-router-dom and next/link.

ğŸ“„ï¸ Layout Routes

To render shared navigation elements like a header, tab bar, or drawer, you can use a layout route.

ğŸ“„ï¸ Layouts

Layouts are parent components that provide a shared UI-driven navigation experience across multiple screens.

ğŸ“„ï¸ Dynamic Routes

Dynamic routes match any unmatched path at a given segment level. For example, /blog/[id] is a dynamic route. The variable part ([id]) is called a "slug". Routes with higher specificity will be matched before a dynamic route. For example, /blog/bacon will match blog/bacon.js before blog/[id].js.

ğŸ“„ï¸ Screen Options

Qualified layouts, like the ones found in expo-router (Stack, Tabs, Drawer, Layout) have a static Screen component which can be used to configure the behavior of a route declaratively.

The global React Navigation ` is managed by Expo Router, you can pass it props like theme and initialState from any screen by using the ` component.

Native apps don't have a server so there are technically no 404s. However, if you're implementing a router universally, then it makes sense to handle missing routes. This is done automatically for each app, but you can also customize it.
